**一、文档说明**

本文档为妙巧巧（山西）技术发展有限公司厂区智能物资定位管理系统的完整功能设计方案。系统以飞书小程序形态开发交付，运行在飞书客户端内置的小程序容器中，面向厂区内8种核心角色，解决「物品在哪里」这一高频需求。

本方案的核心原则是：每一个功能的设计，都必须明确追溯到前期分析的六层痛点中的具体条目。不设计任何没有对应痛点的功能，确保每一行代码都在解决真实问题。

**二、角色与权限体系设计**

**2.1 权限分级架构**

系统采用三级权限架构：查看权限、编辑权限、管理权限。不同角色默认授予不同级别的权限，管理层可自定义调整。角色识别基于飞书通讯录API自动完成，无需手动配置。

  ------------------- ------------------------ ---------------- -------------- ------------------------
  **角色**            **查看范围**             **编辑权限**     **管理权限**   **特殊限制**

  实验室检测人员      全厂区（管控品需授权）   实验室区域物品   无             易制毒品仅看到领用入口

  仓储/物料管理人员   全厂区                   所有库区物品     库位管理       无

  品控/质检人员       全厂区                   品控工具借还     无             无

  犬舍管理/饲养员     犬舍+成品区+库房         犬舍区域物品     无             按犬只编号过滤

  行政/后勤人员       办公区+公共区域          行政物品         无             无

  工程/设备维护人员   全厂区                   设备及备件       设备台账       无

  管理层/厂长         全厂区（含管控品）       全部             系统配置       审计日志查看

  新员工（培训期）    全厂区（管控品除外）     无               无             仅查看+引导模式
  ------------------- ------------------------ ---------------- -------------- ------------------------

*痛点解决 → 【第五层-安全合规】新员工入职初期不应看到管控物品位置信息，权限分级解决信息泄露风险*

*痛点解决 → 【第五层-安全合规】易制毒品储存室信息仅对授权的实验室人员和管理层可见，平衡便利性与安全管控*

*痛点解决 → 【第二层-软件使用】不同角色看到不同的默认视图，减少信息过载，降低使用门槛*

**三、飞书小程序技术适配说明**

系统以飞书小程序形态开发交付，运行在飞书客户端内置的小程序容器中。相比H5网页应用，飞书小程序具备调用原生设备能力（扫码、拍照、录音、本地缓存等）的关键优势，直接影响多项核心功能的实现质量。

**3.1 技术形态优势**

飞书小程序相比H5网页应用的核心优势包括：可调用飞书原生API（扫码、拍照、录音、定位等）；天然集成飞书身份认证体系，无需额外登录流程；支持本地数据缓存，弱网环境下仍可访问核心数据；可添加到飞书工作台和聊天快捷入口，触达路径更短；启动速度优于H5网页，紧急场景下响应更快。

*痛点解决 → 【第六层-角色7工程人员】紧急维修时需要极快响应------小程序启动速度优于H5，3秒内可进入搜索*

*痛点解决 → 【第六层-角色4品控人员】弱网环境（如冷库附近）无法使用------本地缓存确保核心功能可用*

**3.3 应用发布与部署要求**

飞书小程序需要在飞书开放平台创建应用、配置权限范围、提交审核后才能在企业内使用。需要企业管理员在飞书管理后台完成以下前置工作：

▸ 在飞书开放平台创建企业自建应用，选择「小程序」作为应用类型

▸ 配置小程序的可用范围（全员或指定部门）

▸ 授权所需的API权限（扫码、通讯录、审批、消息卡片等）

▸ 配置审批模板（用于管控物品领用流程）

▸ 创建应用机器人并添加到相关群组

▸ 将小程序添加到飞书工作台和聊天「+」菜单快捷入口

**3.4 离线缓存机制**

利用飞书小程序的本地存储能力（tt.setStorage），在用户首次加载后将以下数据缓存到本地：厂区平面图数据、物品分类树、用户个人的常用物品和收藏列表、最近搜索记录。当网络不佳时（如冷库附近信号弱），小程序自动切换到缓存模式，显示本地数据并标注「离线数据，仅供参考」。网络恢复后自动同步最新数据。

*痛点解决 → 【第二层-角色4品控人员】弱网环境下系统无法使用------离线缓存确保核心功能可用*

*痛点解决 → 【第六层-角色3仓管】在冷库里操作时信号差------本地缓存保障基本查询能力*

**四、核心功能模块一：智能搜索引擎**

搜索是系统的第一入口，也是所有角色使用频率最高的功能。搜索引擎的设计目标是：让任何人用任何方式描述一个物品，都能在3秒内找到它。

**4.1 模糊语义搜索**

系统支持口语化、别名化的搜索输入。用户输入「双面胶」可以匹配到系统中名为「泡棉胶带」的物品；输入「那个量血的机器」可以匹配到「血常规分析仪」。后台建立物品别名库和语义关联库，支持拼音搜索、缩写搜索、口语化搜索。

*痛点解决 → 【第二层-角色6行政人员】不知道物品在系统里的标准名称，搜不到就放弃------模糊搜索消除命名壁垒*

*痛点解决 → 【第二层-角色3仓管】别人用口语化描述来问物品位置，精确搜索找不到------语义匹配解决口语化查询*

*痛点解决 → 【第二层-角色9新员工】不了解物品命名规范，不知道该搜什么关键词------别名库+拼音搜索降低搜索门槛*

**4.2 按犬只编号关联搜索**

饲养员输入犬只编号（如K-027），系统自动列出该犬只对应的所有专属物品位置：定制犬粮（成品间/库房）、健康档案（实验室/办公区）、专属标签等。搜索结果按物品类型分组展示。

*痛点解决 → 【第一层-角色5饲养员】必须精准匹配犬只与专属犬粮，找错会影响业务数据------按犬只编号搜索确保一一对应*

*痛点解决 → 【第二层-角色5饲养员】系统只能按物品类别查找，不支持按犬只关联------犬只维度搜索直击核心需求*

**4.3 语音搜索**

在小程序内直接调用tt.getRecorderManager录音，录音完成后调用语音识别服务将语音转为文字，再进入模糊搜索流程。用户长按麦克风图标说出物品名称即可搜索。底层走飞书小程序原生录音能力，稳定性远优于H5的WebRTC方案。同时保留飞书群聊发语音给机器人的备选通道。

*痛点解决 → 【第二层-角色2实验室人员】双手戴手套、沾试剂时无法操作手机打字------语音搜索解放双手*

*痛点解决 → 【第六层-角色4品控人员】巡检时边走边查，需要快速搜索------语音输入比打字更适合移动场景*

**4.4 常用物品快捷入口**

系统根据每个用户的搜索历史，自动在首页展示「最近搜索」和「常找物品」快捷入口。用户也可手动收藏高频物品，一键直达位置信息。

*痛点解决 → 【第二层-角色4品控人员】频繁查找同一类检测工具，每次重新搜索效率低------收藏+历史记录提供快捷通道*

*痛点解决 → 【第六层-角色6行政人员】使用低频，每次打开忘了怎么操作------历史记录帮用户快速回到上次状态*

**4.5 分类浏览模式**

除搜索外，系统提供按区域浏览（生产区/实验室/仓储区/办公区/犬舍）和按类别浏览（原材料/试剂耗材/设备工具/包装材料/办公用品/犬粮成品）两种导航方式。每个分类下展示物品列表及其位置。

*痛点解决 → 【第二层-角色9新员工】不知道该搜什么，搜索框对新人形成使用障碍------分类浏览提供「发现式」导航*

*痛点解决 → 【第一层-角色9新员工】对厂区完全陌生，靠记忆学习效率低------按区域浏览帮助建立厂区空间认知*

**五、核心功能模块二：厂区可视化地图**

基于妙巧巧厂区平面图（总面积约1012.39平方米）构建交互式数字地图，将抽象的「房间名」转化为可点击、可导航的空间视图。地图数据通过离线缓存存储在本地，弱网环境下仍可正常浏览。

**5.1 交互式厂区平面图**

将工厂平面图数字化为可缩放、可平移的交互式地图。地图上标注所有功能区域：备料间、清洗间、乳化间、蒸煮间、灌装间、成品间、实验室、微生物实验室、易制毒品储存室、冷库、小库房、库房、犬舍、办公室1/2/3、文印室、工程部、前台、培训室、茶水间等。点击任意区域可展开该区域内的物品列表。

*痛点解决 → 【第一层-角色9新员工】对厂区布局不熟悉，被告知物品在某个房间但不知道房间在哪------地图直观展示空间位置*

*痛点解决 → 【第二层-角色4品控人员】知道物品在哪个房间但不知道怎么走过去------地图提供相对位置参考*

**5.2 物品位置标注与闪烁定位**

搜索到物品后，地图自动跳转并高亮闪烁该物品所在的区域和具体位置点。如果物品有精确位置信息（如「工程部-A柜-第3层」），地图上对应区域会展示局部放大视图和文字标注。

*痛点解决 → 【第二层-全角色共性】信息颗粒度不匹配------地图支持从区域到房间到柜子的多层级定位*

*痛点解决 → 【第一层-角色7工程人员】知道备件在工程部但不知道具体柜子和抽屉------局部放大展示精确位置*

**5.3 区域状态着色**

管理层视图下，地图上各区域按状态着色：绿色=数据已更新（7天内）、黄色=数据较旧（7-30天）、红色=数据过期（超30天未更新）。让管理层一眼看到哪些区域的数据需要维护。

*痛点解决 → 【第三层-角色8管理层】无法判断系统数据是否最新------区域着色提供数据可信度的直观指标*

*痛点解决 → 【第二层-角色8管理层】缺乏全局视图、无法远程掌握物资状态------着色地图是全局概览的最直接载体*

**5.4 动线导航建议**

当用户需要连续找多个物品时（如品控巡检需要依次到备料间、蒸煮间、灌装间取样），系统根据厂区平面图计算最优行走路线，标注在地图上，并显示预计步行时间。

*痛点解决 → 【第一层-角色4品控人员】跨区域流动作业，需要同时追踪多个位置------动线规划减少跑冒枉路*

*痛点解决 → 【第一层-角色3仓管】冷库、小库房、库房三个空间分散，来回跑耗时------路线优化减少无效跑动*

**六、核心功能模块三：物品信息管理**

物品是系统的核心数据实体。每个物品条目需要承载位置信息、基本属性、关联关系等多层数据。

**6.1 物品信息卡片**

每个物品拥有一张信息卡片，包含以下字段：物品名称、别名（支持多个）、所属分类、当前位置（区域\>房间\>具体位置）、位置更新时间、负责人/归属部门、物品状态（在库/借出/使用中/维修中）、关联犬只编号（如适用）、有效期/保质期（如适用）、照片（支持拍照上传）、备注。

*痛点解决 → 【第一层-角色2实验室人员】试剂品类多且持续增加，有效期管理压力大------信息卡片包含有效期字段，到期自动提醒*

*痛点解决 → 【第一层-角色5饲养员】需要精准匹配犬只与专属犬粮------信息卡片关联犬只编号字段*

*痛点解决 → 【第三层-角色9新员工】录入数据不规范会污染数据库------信息卡片提供标准化字段模板*

**6.2 物品分类体系**

系统预设六大一级分类：原材料（肉类/碳水/蔬菜/调味料）、试剂耗材（ELISA试剂盒/生化试剂/通用耗材）、设备工具（生产设备/检测仪器/维修工具）、包装材料（包装袋/标签/纸箱）、办公用品（文具/打印耗材/清洁工具/劳保用品）、犬粮成品（民用版/军警版）。分类可由管理员自定义扩展。

*痛点解决 → 【第二层-角色6行政人员】不知道物品归在哪个分类下------预设分类体系+搜索联想消除分类困惑*

*痛点解决 → 【第三层-角色6行政人员】杂项物品从未被系统性盘点归类------预设分类模板降低初始化门槛*

**6.3 设备台账管理**

针对角色7（工程/设备维护人员）的特殊需求，系统为每台设备建立独立台账。台账信息包括：设备名称与型号、设备编号、安装位置、购置日期、上次维修日期、维修记录列表、关联备件列表（每个备件是独立的物品条目，标注存放位置）、设备状态（正常/故障/维修中/停用）。从设备台账可一键跳转到该设备所有备件的位置信息。

*痛点解决 → 【第一层-角色7工程人员】设备分布广、零件和工具存放位置分散------设备台账将设备与备件关联，一键定位所有相关物品*

*痛点解决 → 【第二层-角色7工程人员】系统只显示「备件在工程部」但不知道哪个柜子------台账中备件精确到柜/层/抽屉*

*痛点解决 → 【第五层-角色7工程人员】特种设备（如全自动杀菌锅）维修记录需存档备查------台账维修记录满足合规要求*

**6.4 批次与追溯管理**

针对仓管和品控的需求，系统支持为同一类物品区分不同批次。每个批次记录入库时间、供应商、数量、有效期、对应库位。支持按批次查找和按时间排序，优先展示即将过期的批次。特别针对个性化犬粮成品，每一份犬粮按犬只编号和配方编号唯一标识，支持从犬只→犬粮→原材料批次的正向追溯和从原材料批次→犬粮→犬只的逆向追溯。

*痛点解决 → 【第一层-角色3仓管】SKU多但每个量小、批次多周转快------批次管理解决小批量多批次的库位混乱*

*痛点解决 → 【第五层-角色3仓管】原材料批次追溯是食品安全合规要求------批次追溯链支持审计*

*痛点解决 → 【第五层-角色5饲养员】犬粮喂养记录需要准确------犬粮批次与犬只关联确保可追溯*

**七、核心功能模块四：数据维护与更新机制**

数据维护是系统生死存亡的关键。设计原则是：让维护的成本低于直接问人的成本，让数据更新成为工作流的自然组成部分而不是额外负担。

**7.1 扫码快速更新**

为每个重要库位、柜子、货架生成专属二维码并打印粘贴。当物品被放入或取出某个位置时，相关人员在小程序内点击「扫码」按钮，调用tt.scanCode调起飞书原生扫码界面，扫描库位二维码后自动跳转到该库位页面，选择要存入/取出的物品，完成更新。支持连续扫码模式：扫完库位码后直接扫物品码，两码关联自动完成位置绑定。整个操作控制在2步以内，耗时不超过5秒。

*痛点解决 → 【第三层-角色3仓管】出入库频率高，每次更新如果超过3步就会被跳过------扫码更新控制在2步以内*

*痛点解决 → 【第三层-角色2实验室人员】每次移动试剂都要打开系统更新太重------扫码更新比手动输入快10倍*

*痛点解决 → 【第六层-角色3仓管】在库房里搔货时打开飞书不是自然动作------扫码操作比打开应用搜索更符合库房场景*

**7.2 拍照更新位置**

对于没有条码的物品（如维修工具、办公用品），调用tt.chooseImage拍照或从相册选择，拍照后图片直接上传到服务端与物品位置记录关联。飞书小程序原生拍照体验流畅，可作为位置更新的常规操作方式。照片同时作为位置参考留存，方便其他人直观了解物品的具体摆放状态。

*痛点解决 → 【第三层-角色7工程人员】备件和工具没有条码，录入精确位置太繁琐------拍照比文字描述更快更准*

*痛点解决 → 【第二层-角色7工程人员】系统只告诉「在工程部A柜」但不知道具体哪层------照片直观展示实际摆放位置*

**7.3 数据新鲜度机制**

系统为每条位置数据标注「最后确认时间」。超过设定周期（默认7天）未确认的数据标记为「待确认」，超过30天标记为「可能过期」。系统定期通过飞书消息卡片向各区域负责人推送待确认清单，负责人可直接在卡片上点击「位置正确」或「需要更新」按钮完成确认，无需打开小程序。

*痛点解决 → 【第三层-角色8管理层】无法判断数据可信度------新鲜度标签让每条数据的可靠程度透明可见*

*痛点解决 → 【第二层-全角色共性】查到旧位置比没有数据更糟------过期标记防止用户依赖失效信息*

*痛点解决 → 【第三层-全角色共性】数据腐烂导致系统被弃用------定期确认机制以极低成本保持数据鲜活*

**7.4 众包式数据维护**

任何用户在查找物品后，如果发现实际位置与系统记录不符，可以一键提交「位置纠错」。纠错信息推送给该物品的负责人审核确认。新员工在发现物品位置时也可以提交「位置发现」，由老员工审核后入库，让新员工从数据的消费者变成贡献者。

*痛点解决 → 【第三层-角色4品控人员】品控工具被带到别的车间没还，但不觉得自己该更新------任何人发现异常都可纠错*

*痛点解决 → 【第三层-角色9新员工】不允许维护数据会浪费一线信息------审核机制下新员工也能安全贡献数据*

*痛点解决 → 【第四层-角色3仓管】被反复打扰帮别人找东西------如果别人自己能查到+纠错，仓管被打扰的频率大幅下降*

**7.5 初始化数据导入**

系统提供Excel模板批量导入功能，支持按区域分批导入物品数据。导入模板预填物品分类和标准字段，降低首次录入的门槛。同时提供「渐进式建库」模式：不要求一次录入所有物品，可以先录入高频使用的100个物品，后续在日常使用中通过扫码、拍照、众包纠错等方式逐步补充。系统统计录入完成度并定期提醒。

*痛点解决 → 【第三层-角色6行政人员】初始录入工作量巨大，杂项物品从未被系统盘点------Excel导入+渐进式建库降低初始化门槛*

*痛点解决 → 【第三层-角色7工程人员】备件和工具颗粒度极细，全量录入不现实------先录高频物品，渐进补充*

*痛点解决 → 【第三层-角色2实验室人员】14项扩展到27项后试剂种类翻倍------批量导入应对物品数量快速增长*

**八、核心功能模块五：跨角色协同与流转**

物品位置不是孤立信息，它天然涉及跨部门、跨角色的协同。本模块聚焦解决第四层管理与协同痛点。

**8.1 物品流转记录**

系统自动记录每个物品的位置变更历史，形成完整的流转链条。每条记录包含：变更时间、变更人、从哪里到哪里、变更原因（领料/归还/转移/维修等）。当物品找不到时，查看流转记录即可知道最后是谁动了它、什么时候、放到了哪里。

*痛点解决 → 【第四层-全角色共性】东西找不到了不知道是谁没放好还是谁用了没还------流转记录解决归责问题*

*痛点解决 → 【第四层-角色4品控/角色7工程】跨区域取用物品后其他人不知道------流转记录自动同步*

*痛点解决 → 【第五层-角色2实验室/角色3仓管】合规审计需要完整的物品流转链------自动记录满足审计追溯*

**8.2 领用与归还通知**

当某个物品被标记为「借出」或「领用」时，系统自动通过飞书消息卡片向该物品的负责人发送通知。归还时同样通知。通知内容包含借用人、预计归还时间、物品当前位置。超过预计归还时间未归还的物品，系统自动发送催还提醒。

*痛点解决 → 【第四层-角色4品控人员】品控取走检测工具后该区域负责人不知道------自动通知消除信息盲区*

*痛点解决 → 【第四层-角色7工程人员】维修时移动了其他物品，区域负责人不知情------通知机制确保知情权*

**8.3 物料交接确认**

当物品从一个区域/角色转移到另一个区域/角色时（如试剂从库房转移到实验室），系统要求交接双方确认。移出方点击「已移出」，接收方点击「已接收」，双方确认后位置自动更新。如果只有一方确认超过2小时，系统提醒另一方。

*痛点解决 → 【第四层-角色2实验室与角色3仓管之间】试剂从库房转移到实验室，到底谁更新位置------双方确认机制明确责任*

*痛点解决 → 【第四层-角色5饲养员与生产人员之间】不知道犬粮做好了没有、放在哪里了------交接确认同步生产进度和位置信息*

**8.4 物品状态看板**

每个区域提供一个状态看板，展示：该区域内所有物品的数量和分类统计、借出中的物品列表（谁借的、什么时候借的）、近期入库的物品列表、即将过期的物品列表。各区域负责人可以在看板上一目了然掌握本区域的物资状态，无需逐个搜索。

*痛点解决 → 【第四层-角色8管理层】信息获取全靠问人------区域看板提供按区域的物资汇总，无需拼凑碎片信息*

*痛点解决 → 【第一层-角色3仓管】库位记忆负担重------看板实时展示各库区物品分布，减少记忆依赖*

**九、核心功能模块六：安全与合规管理**

公司持有动物诊疗许可证，设有易制毒品储存室，生产犬粮涉及食品安全。合规不是加分项，而是生命线。本模块聚焦解决第五层安全与合规痛点。

**9.1 管控物品专项管理**

易制毒品储存室中的物品在系统内标记为管控物品。管控物品的查看权限仅开放给授权的实验室检测人员和管理层。管控物品的每一次查看、领用、归还都自动记录日志，日志包含操作人、操作时间、操作类型、数量变化。日志不可修改、不可删除，支持按时间范围导出为PDF/Excel用于合规审计。

*痛点解决 → 【第五层-角色2实验室人员】易制毒品有双人双锁、领用登记的法规要求------电子台账替代纸质登记*

*痛点解决 → 【第五层-角色2实验室人员】系统让所有人看到管控品位置是安全隐患------权限隔离+日志审计平衡便利与安全*

*痛点解决 → 【第五层-角色8管理层】审查时需要快速调取完整记录------日志一键导出为审计材料*

**9.2 操作审计日志**

系统对所有物品的增删改操作记录完整的审计日志。日志字段包括：操作时间、操作人、操作类型（新增/修改/删除/查看/导出）、操作对象、变更前后的值。日志存储独立于业务数据，采用追加写入模式，确保不可篡改。管理层可在后台按时间、人员、操作类型筛选查看。

*痛点解决 → 【第五层-角色4品控人员】品控操作记录可能被视为质量管理辅助证据------审计日志确保数据完整性和可追溯性*

*痛点解决 → 【第五层-角色8管理层】需要对系统数据的可靠性有信心------审计日志让所有操作有据可查*

**9.3 有效期预警**

对有有效期属性的物品（试剂、原材料、犬粮成品等），系统设置三级预警：距过期30天（提醒关注）、距过期7天（紧急提醒）、已过期（红色警告）。预警通过飞书消息卡片推送给物品负责人和管理层。过期物品在搜索结果中以醒目标记展示，防止误用。

*痛点解决 → 【第一层-角色2实验室人员】ELISA试剂盒有严格有效期，找到了但过期了等于没找到------三级预警确保试剂在有效期内被使用*

*痛点解决 → 【第五层-角色3仓管】肉类食材冷链存储有食品安全要求------过期预警防止过期原材料进入生产线*

**9.4 权限操作留痕**

所有权限变更操作（授权/取消授权/修改角色）均记录日志。新员工转正时权限升级、员工离职时权限回收，都自动记录并通知管理层确认。

*痛点解决 → 【第五层-角色9新员工】权限分级避免培训期接触敏感信息------权限变更留痕确保管理可控*

**十、核心功能模块七：飞书深度集成**

系统以飞书小程序形态交付，深度集成意味着系统的入口、通知、交互都融入飞书的工作流中，让使用系统成为使用飞书的自然延伸。

**10.1 飞书工作台+聊天双入口**

小程序同时配置为飞书工作台应用图标和聊天会话「+」菜单快捷入口。工作台入口用于日常主动查询，聊天入口用于对话中临时查找。小程序首次打开通过tt.login获取用户身份，与飞书通讯录API联动自动识别角色和权限，全程免登录。同时配置三个小程序快捷入口：扫码（直接调起扫码）、搜索（直接聚焦搜索框）、地图（直接展示厂区平面图）。

*痛点解决 → 【第六层-角色6行政人员】入口埋太深就不会用------工作台+聊天双入口最大化减少触达路径*

*痛点解决 → 【第二层-角色7工程人员】紧急维修时要登录、选部门、再搜索太慢------免登录+自动识别角色+扫码快捷入口*

**10.2 飞书机器人主动推送（交互式卡片）**

系统配置飞书机器人，主动向不同角色推送关键信息。推送内容包括：每日物资概览（推送给管理层）、待确认位置数据清单（推送给各区域负责人）、有效期预警（推送给物品负责人）、借出超期催还提醒（推送给借用人）、新入库物品通知（推送给相关角色）。所有推送采用飞书交互式消息卡片，用户可直接在卡片上点击「位置正确」「需要更新」等按钮完成操作，简单确认无需打开小程序，点击「需要更新」则跳转到小程序编辑页面。

*痛点解决 → 【第六层-角色8管理层】不愿意主动打开系统查信息------机器人主动推送让管理层被动接收关键数据*

*痛点解决 → 【第三层-全角色共性】数据确认容易被遗忘------定期推送提醒+卡片内直接确认，降低操作门槛*

*痛点解决 → 【第六层-角色5饲养员】日常不怎么用飞书也不会主动打开系统------推送消息在飞书消息流中触达*

**10.3 飞书群聊快捷查询**

在飞书群聊中@系统机器人+物品名称，即可在群内直接返回物品位置信息卡片。例如在生产群里发「@找物助手 斛拌机备用刀片」，机器人直接回复位置信息卡片（精简为文字+跳转链接形式，适配群聊展示空间）。无需切换到小程序。

*痛点解决 → 【第六层-角色7工程人员】紧急维修时需要极快响应------群聊内直接查询比切换应用更快*

*痛点解决 → 【第六层-角色4品控人员】巡检时来回切换应用影响工作节奏------群聊查询不打断当前对话*

*痛点解决 → 【第四层-角色3仓管】被反复打扰帮别人找东西------群里有机器人回答，仓管不用再充当人肉搜索引擎*

**10.4 飞书审批流打通**

管控物品的领用申请接入飞书审批流。实验室人员在小程序内发起领用申请，审批单自动流转到管理层的飞书审批列表。审批通过后系统自动更新物品状态和位置。

*痛点解决 → 【第五层-角色2实验室人员】管控物品查找后还要切换到别的系统做登记------审批流打通避免系统切换*

*痛点解决 → 【第六层-角色2实验室人员】系统独立于已有工作流是额外入口------嵌入飞书审批让合规操作成为工作流的自然环节*

**十一、核心功能模块八：新人引导与培训系统**

新员工是最需要这套系统的群体，但也最容易被劵退。本模块专门为角色9设计，同时服务于角色5（信息化操作习惯较弱的饲养员）。

**11.1 首次使用引导流程**

新用户首次打开小程序时，自动进入引导模式。引导流程包括：厂区地图概览（交互式浏览每个区域的名称和用途）、搜索示范（引导用户完成一次搜索操作）、分类浏览示范（展示如何按区域/类别查看物品）、扫码体验（引导扫描一个示范二维码）。整个引导控制在3分钟以内。引导可随时跳过，后续可在设置中重新触发。

*痛点解决 → 【第二层-角色9新员工】面对搜索框不知道从哪下手------引导流程手把手教完成第一次操作*

*痛点解决 → 【第二层-角色5饲养员】界面太复杂用几次就不想用了------引导降低初次使用的认知负担*

*痛点解决 → 【第六层-角色9新员工】培训一次但几天后就忘了------可重复触发的引导替代一次性培训*

**11.2 上下文操作提示**

在用户操作过程中，系统根据用户行为提供实时提示。例如：搜索无结果时提示「试试换个关键词或使用分类浏览」；查看物品位置时提示「在地图上查看」；位置信息显示「待确认」时提示「此信息已超过7天未确认，建议到现场核实」。提示以非侵入式的浮层展示，不阻断主操作流程。

*痛点解决 → 【第二层-角色9新员工】不知道系统里物品叫什么------搜索无结果时的引导防止用户直接放弃*

*痛点解决 → 【第二层-角色6行政人员】使用频率低，每次打开忘了怎么操作------上下文提示在需要时才出现，不增加记忆负担*

**11.3 厂区认知测验（可选）**

面向新员工提供可选的厂区认知小测验。测验以交互式地图为基础，提问如「斛拌机在哪个车间？」「冷库在厂区的哪个方位？」。答对后该区域在用户的个人地图上点亮。这是一个游戏化的学习机制，帮助新员工快速建立厂区空间记忆。

*痛点解决 → 【第一层-角色9新员工】靠记忆学习厂区布局效率低------交互式测验通过主动回忆加深空间记忆*

*痛点解决 → 【第四层-角色9新员工与老员工之间】新人频繁问老员工，造成双向效率损失------系统替代人肉搜索引擎*

**十二、核心功能模块九：管理驾驶舱**

管理层不需要逐个搜索物品，他们需要的是全局视图和关键指标。管理驾驶舱是专门为角色8设计的功能模块。

**12.1 全局物资概览**

以数字卡片+厂区着色地图的形式展示：全厂物品总数及各分类占比、各区域物品数量分布、数据新鲜度总览（已确认/待确认/可能过期的比例）、当前借出物品数量、即将过期物品数量、本周新增/消耗物品统计。

*痛点解决 → 【第一层-角色8管理层】信息获取全靠问人、缺乏全局视图------驾驶舱一页展示全局物资状态*

*痛点解决 → 【第二层-角色8管理层】系统只能找单个物品不能看全局------驾驶舱提供宏观视图*

**12.2 异常预警中心**

汇总所有需要管理层关注的异常事项：过期物品未处理、管控物品异常变动、数据长期未更新的区域、借出超期未还的物品、权限变更待审批。每条异常可一键跳转到详情页处理。

*痛点解决 → 【第二层-角色8管理层】打开系统只能一个个搜，无法发现异常------异常预警主动暴露问题*

*痛点解决 → 【第五层-角色8管理层】合规检查时需要快速发现和解决问题------异常中心集中展示风险点*

**12.3 数据质量报表**

按周/月生成数据质量报表，统计：各区域的数据更新率、各角色的数据维护参与度、纠错记录数量和处理率、物品录入完成度（已录入vs估计应有）。报表支持导出PDF，可用于内部管理会议和外部审计。

*痛点解决 → 【第三层-角色8管理层】没有办法判断系统数据是不是最新的------数据质量报表量化数据可信度*

*痛点解决 → 【第四层-角色8管理层】需要知道各区域物资管理状态------报表按区域/角色维度汇总协同效率*

**十四、开发优先级建议**

基于功能的价值密度（解决痛点数量）和开发复杂度，建议按三个阶段推进。由于采用飞书小程序技术路线，扫码和语音能力可原生调用，开发成本显著降低，建议将扫码和语音搜索提前到第一阶段。

**第一阶段：MVP最小可用版本（建议4-6周）**

核心目标：让系统能用起来，解决最高频的「找东西」需求。

+--------------------------------------------------------------------------------------+
| **前置工作**                                                                         |
|                                                                                      |
| 在飞书开放平台创建应用、配置权限、创建机器人、配置审批模板、添加工作台和聊天快捷入口 |
+--------------------------------------------------------------------------------------+

▸ 智能搜索引擎（模糊搜索+分类浏览+语音搜索）

▸ 厂区可视化地图（交互式平面图+物品位置标注）

▸ 物品信息管理（物品卡片+分类体系）

▸ 扫码快速更新（调用tt.scanCode原生扫码）

▸ 飞书工作台+聊天双入口+免登录

▸ 初始化数据导入（Excel模板）

▸ 首次使用引导流程

▸ 离线缓存机制（厂区地图+高频物品本地存储）

*痛点解决 → 本阶段覆盖：第一层80%的业务痛点 + 第二层70%的使用痛点 + 第六层40%的融入痛点*

**第二阶段：协同与维护版本（建议6-8周）**

核心目标：让数据活起来，让多角色协同起来。

▸ 拍照更新位置（调用tt.chooseImage原生拍照）

▸ 数据新鲜度机制+众包纠错

▸ 物品流转记录+领用归还通知

▸ 物料交接确认

▸ 飞书机器人推送（交互式卡片）+群聊快捷查询

▸ 区域状态看板

▸ 按犬只编号关联搜索

*痛点解决 → 本阶段覆盖：第三层90%的维护痛点 + 第四层80%的协同痛点 + 第六层70%的融入痛点*

**第三阶段：合规与管理版本（建议4-6周）**

核心目标：满足合规要求，赋能管理决策。

▸ 管控物品专项管理+操作审计日志

▸ 有效期预警系统

▸ 设备台账管理

▸ 批次与追溯管理

▸ 管理驾驶舱（全局概览+异常中心+质量报表）

▸ 飞书审批流打通

▸ 厂区认知测验

▸ 动线导航建议

▸ 权限操作留痕

*痛点解决 → 本阶段覆盖：第五层100%的合规痛点 + 管理驾驶舱全部功能*
